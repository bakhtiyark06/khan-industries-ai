## Executive Summary

Khan Industries requires its first production website — the company's **digital headquarters** — to replace the default Next.js starter page and establish a credible, premium public presence. This PRD governs **Phase 1 — Foundation** only, as defined in `04_WEBSITE.md` §12.

**What is being built:** A fast, accessible, dark-first marketing website with Home, About, Contact, Solutions hub (with four priority service detail pages), Products hub (with honest Concept-status product pages), design tokens and reusable components, SEO baseline, and a consultation request form wired to email notification (CRM stub).

**Who it serves:** Prospective clients (small business owners, startups, enterprise evaluators) who need to understand Khan Industries, browse offerings honestly, and request a consultation within sixty seconds of landing.

**Why now:** KOS v1.0 is complete. The company's first major business goal (`00_COMPANY.md`) includes building a professional online presence. Without a production website, the services funnel, product showcase, and consultation capture cannot operate. The website itself demonstrates the Website Development service capability (`03_SERVICES.md` §3.3).

**Deployment approach:** Phase 1 **does not require a purchased custom domain**. The site may launch on a **Vercel-generated deployment URL**. Production Domain remains **TBD Before Production** until a custom domain is purchased. Custom domain migration is a follow-on enhancement (§44), not a Phase 1 gate.

**What is explicitly not included:** Portfolio, Industries pages, Careers, Legal pages, Insights blog, authentication, Client Portal, AI Marketplace, booking integration, newsletter, global search, and product waitlist flows — all deferred to Phase 2+ per `04_WEBSITE.md` §12.

**Timeline:** TBD — founder-led Stage 1 build.  
**Approval status:** Draft — pending founder Product Review.

No invented traction, client logos, testimonials, or domain names appear in this document or in required site content.

## Business Problem

Khan Industries is an early-stage technology company with a complete Khan Industries Operating System (KOS) and a default Next.js starter page as its only digital surface. This creates four organizational problems:

1. **No credible public presence.** Prospects cannot evaluate the company, its capabilities, or its offerings. The starter page undermines premium positioning and the Website Development service promise.
2. **Blocked revenue funnel.** The canonical funnel (`AGENTS.md`, `07_SALES.md`) begins with discovery on the website. Without consultation capture, leads cannot enter the pipeline.
3. **No product showcase.** Ten Concept products (`02_PRODUCTS.md` §4) have no honest public home. Sales conversations lack a reference surface.
4. **No scalable platform foundation.** Future authenticated surfaces (Client Portal, Marketplace, Dashboard) require marketing route architecture today (`04_WEBSITE.md` §10.2, `05_ENGINEERING.md` §7).

**Value outcomes mapped (`00_COMPANY.md`):**

| Outcome | How Phase 1 addresses it |
| --- | --- |
| Increase revenue | Enables consultation → proposal → client funnel |
| Improve quality | Site quality demonstrates execution standard |
| Make systems easier to run | Central hub for products, services, and future client surfaces |

## User Journeys

### Journey 1 — Primary Commercial (`04_WEBSITE.md` §5.1)

```
Home (orient in <60s)
    ↓
Solutions or Products (explore offering)
    ↓
Service or Product detail (understand fit)
    ↓
Contact / Consultation form (submit intent)
    ↓
Founder notification → CRM record → Consultation scheduled
```

**Post-submit expectation (`07_SALES.md` §9.3):** Confirmation message explains next steps, expected response time (TBD — founder sets at launch), and no spam commitment.

### Journey 2 — Product Discovery → Consultation

```
Products hub (scan Concept products with status badges)
    ↓
Product detail page (problem, outcome, honest status)
    ↓
CTA: Request consultation or learn more (no fake "Buy now")
    ↓
Contact form (pre-filled product interest field optional)
```

### Journey 3 — Solutions → Consultation

```
Solutions hub (services by category)
    ↓
Service detail page (e.g., AI Agent Development)
    ↓
Sticky CTA: Book a consultation
    ↓
Contact form
```

### Homepage Strategy

Per `04_WEBSITE.md` Home §4.1:

- **Purpose:** Orient every visitor in seconds — who Khan Industries is, what problems it solves, three best next steps.
- **Required elements:** Value proposition, primary CTA (consultation), secondary paths to Products and Solutions.
- **Trust signals:** Real only — product demos when available, founder credibility on About, premium design execution. No fake logos or testimonials.
- **Layout:** Hero with one primary CTA; featured services and products blocks; concise company proof points.

### Information Architecture (Phase 1 Public Only)

```
/                           Home
/about                      About
/contact                    Contact (consultation form)
/solutions                  Solutions hub
/solutions/[slug]           Service detail (4 priority pages)
/products                   Products hub
/products/[slug]            Product detail (10 Concept products)
```

**Excluded from Phase 1:** `/portfolio`, `/industries`, `/insights`, `/careers`, `/portal`, `/marketplace`, `/developers`, authenticated routes.

### Navigation Structure

**Header (desktop)** per `04_WEBSITE.md` §6.1:

| Item | Phase 1 behavior |
| --- | --- |
| Logo → Home | Yes |
| Solutions | Dropdown or hub link |
| Products | Dropdown or hub link |
| About | Link |
| Contact | Primary CTA button (visually distinct) |
| Insights | **Omitted from header** — no content in Phase 1; footer note only (Decision Log DL-003) |

**Footer:** Company (About, Contact), Solutions (top pages), Products (top concepts), copyright. Legal and Insights links added in Phase 2.

**Mobile:** Hamburger menu; Contact CTA visible without opening menu; 44px touch targets.

### Design Philosophy

- **Dark-first**, spacious, minimal — inspired by Apple, Stripe, Anthropic (`AGENTS.md`, `01_BRAND.md`)
- **Calm confidence** — premium through restraint, not noise
- **Design tokens first** — colors, typography, spacing from `01_BRAND.md` §7–8 before page builds
- **One primary CTA per section** (`04_WEBSITE.md` §9.2)

### Conversion Strategy

| Priority | Path | Phase 1 |
| --- | --- | --- |
| 1 | Consultation | Primary — Contact CTA everywhere |
| 2 | Products | Secondary — explore scalable software |
| 3 | Solutions | Secondary — explore custom services |

**Trust-first rules:** Honest product status; explain post-submit process; no countdown timers, fake scarcity, or dark patterns (`04_WEBSITE.md` §9.3).

### Lead Generation

Primary conversion: **consultation request** via Contact page form. Fields align to `07_SALES.md` §9.1 CRM standards (see Appendix C). Source tracking: `inbound-website` with optional source page hidden field.

### Consultation Flow

```
Visitor completes Contact form
    ↓

    ↓

    ↓
Email notification to founder (provider TBD)
    ↓
Manual CRM entry (spreadsheet or stub) per 07 §9
    ↓
Founder qualifies lead (07 §7)
    ↓
Consultation scheduled (07 §12) if qualified
```

## User Stories

### Layout and Navigation

**US-001** — As a visitor, I want a consistent header and footer on every page, so that I can navigate predictably.

Acceptance:
- [ ] PageLayout wraps all marketing pages
- [ ] Header includes Solutions, Products, About, Contact CTA
- [ ] Footer includes sitemap links
- [ ] Logo returns to Home

**US-002** — As a mobile visitor, I want a usable navigation menu, so that I can reach any page without a mouse.

Acceptance:
- [ ] Hamburger or bottom sheet menu
- [ ] Contact CTA visible in header without opening menu
- [ ] Touch targets ≥ 44px
- [ ] Keyboard and focus trap accessible

### Home

**US-003** — As a first-time visitor, I want to understand Khan Industries within sixty seconds, so that I can decide whether to explore further.

Acceptance:
- [ ] Hero states value proposition clearly
- [ ] Primary CTA links to Contact
- [ ] Secondary links to Products and Solutions
- [ ] No fake social proof

### About

**US-004** — As a prospect, I want to read an honest About page, so that I can assess company credibility.

Acceptance:
- [ ] Mission and values align with `00_COMPANY.md`
- [ ] Early-stage positioning stated honestly
- [ ] No invented history, clients, or awards

### Solutions

**US-005** — As a prospect, I want to browse services by category, so that I can find relevant custom offerings.

Acceptance:
- [ ] Solutions hub lists all nine services from `03_SERVICES.md`
- [ ] Services grouped by category
- [ ] Each card links to detail page or hub anchor

**US-006** — As a prospect, I want to read detail pages for key services, so that I can understand deliverables and outcomes.

Acceptance:
- [ ] Four priority detail pages live (AI Agent Development, AI Chatbots, Website Development, AI Consulting)
- [ ] Each page includes consultation CTA
- [ ] Copy derived from `03_SERVICES.md` — not invented capabilities

### Products

**US-007** — As a prospect, I want to browse product concepts with honest status, so that I know what exists today vs. future.

Acceptance:
- [ ] Products hub lists all ten Concept products from `02_PRODUCTS.md` §4
- [ ] Status badge visible on every product card
- [ ] No product implies Launched

**US-008** — As a prospect, I want to read individual product pages, so that I can evaluate fit and request consultation.

Acceptance:
- [ ] Each product has `/products/[slug]` page
- [ ] Problem, outcome, status, and consultation CTA present
- [ ] No fake pricing or user counts

### Contact

**US-009** — As a prospect, I want to submit a consultation request, so that Khan Industries can follow up.

Acceptance:
- [ ] Form includes required CRM fields (Appendix C)
- [ ] Validation errors shown clearly
- [ ] Success state explains next steps
- [ ] Founder receives email notification

### SEO and Accessibility

**US-010** — As a search engine, I want proper metadata and sitemap, so that pages can be indexed.

Acceptance:
- [ ] Unique title and description per page
- [ ] `sitemap.xml` and `robots.txt` present
- [ ] `metadataBase` matches active production URL

**US-011** — As a user with assistive technology, I want accessible pages, so that I can use the site independently.

Acceptance:
- [ ] Skip to main content link
- [ ] Semantic HTML landmarks
- [ ] WCAG 2.1 AA baseline per `05_ENGINEERING.md` §20

### Engineering

**US-012** — As an engineer, I want route group architecture, so that future auth surfaces attach without rewrite.

Acceptance:
- [ ] `(marketing)/` route group created
- [ ] `src/content/` holds product and service metadata
- [ ] `(app)/` not implemented — structure documented only
